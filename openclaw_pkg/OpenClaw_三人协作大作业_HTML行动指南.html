OpenClaw 三人协作大作业行动指南

选题建议：每日股票分析数字员工系统。目标：三人各部署一个 Claw/数字员工，飞书群触发，大盘、板块、个股分别输出 JSON，组长汇总为日报，并用 HTML 完成最终展示。

1. 作业目标：先跑通，再美化

你们不要一开始就追求复杂模型。最稳路线是：飞书群触发 → 三个 Python 脚本输出 JSON → 汇总机器人生成 HTML 日报 → 云服务器运行 → HTML 页面展示。

必须

三人三员工

每人对应一个数字员工，分别负责大盘、板块、个股。

必须

云服务器部署

最终至少把汇总系统、HTML 页面或机器人服务部署到云服务器。

展示

不用 PPT，用 HTML

把这个 HTML 或你们自己改好的展示页作为答辩页面，配合飞书群实机演示。

3. 总体流程

  1. 三个人分别创建自己的飞书机器人，各自获得 App ID 和 App Secret。
  2. 每个人在自己电脑或服务器上运行 openclaw onboard，渠道选择 Feishu，模型选择 DeepSeek。
  3. 三个机器人加入同一个飞书群。
  4. 群里分别发送：@大盘机器人 运行大盘、@板块机器人 运行板块、@个股机器人 运行个股。
  5. 三个机器人输出 JSON，并保存到 data/reports/。
  6. 组长发送 @汇总机器人 运行汇总，生成最终 HTML 日报。
  7. 答辩时打开 HTML 展示页，展示架构、运行结果、飞书群消息、最终日报和云服务器。

4. 每个人本机安装步骤

4.1 安装基础软件

  • Python 3.10 或 3.11
  • Git
  • VS Code
  • 浏览器：Chrome / Edge

4.2 创建项目文件夹

mkdir openclaw-homework cd openclaw-homework

4.3 安装 OpenClaw

安装命令以班级群公告为准。下面只是常见写法。

pip install openclaw openclaw --help

4.4 初始化 OpenClaw

openclaw onboard
渠道Feishu
模型DeepSeek
API Key组长提供，所有人可共用；不要发到公开仓库
飞书 App ID/Secret每个人填自己的机器人凭证

5. 飞书机器人创建步骤

  1. 进入飞书开放平台，创建“企业自建应用”。
  2. 应用名称分别命名为：大盘分析机器人、板块分析机器人、个股分析机器人。
  3. 在应用凭证页复制 App ID 与 App Secret。
  4. 开启机器人能力，并配置接收群聊消息事件。
  5. 检查权限：读取群消息、发送群消息、机器人进群。
  6. 发布或启用应用，把机器人拉入你们三人的飞书群。
  7. 在群里测试：@机器人 你好。若机器人有回应，说明通道基本成功。

注意：App Secret 和 DeepSeek API Key 不能截图发到报告里，也不能上传到 GitHub。

7. 起步代码与目录

建议使用这个目录结构。起步包里已经放好了 4 个脚本，先保证能跑通；最终再把演示数据替换成真实数据。

openclaw-homework/ ├── agents/ │ ├── common.py │ ├── market_agent.py │ ├── sector_agent.py │ ├── stock_agent.py │ └── summary_agent.py ├── data/ │ └── reports/ ├── ui/ ├── requirements.txt ├── .env.example └── README.md

本地测试顺序

python agents/market_agent.py python agents/sector_agent.py python agents/stock_agent.py python agents/summary_agent.py

看到 JSON 和最终日报，就说明最小闭环已经跑通。

9. 云服务器部署

小白建议只买一台轻量云服务器，由组长部署完整项目；三个人本机也要能运行自己的 Claw，满足“每人部署一个”的要求。

服务器推荐

  • Ubuntu 22.04
  • 2 核 2G 或 2 核 4G
  • 带公网 IP
  • 安全组开放 22、80、必要时开放 7860

部署命令

ssh root@你的服务器IP apt update apt install python3 python3-pip git screen -y git clone 你的仓库地址 cd openclaw-homework pip install -r requirements.txt screen -S openclaw openclaw run

按 Ctrl + A，再按 D，可以让程序留在后台运行。

HTML 页面发布最简方案

# 在服务器项目目录启动静态网页 cd data/reports python3 -m http.server 8000 # 浏览器访问：http://服务器IP:8000/日期_final_report.html

10. 报告与提交材料

  • 实验报告 PDF：包含整体思路、三人分工、数据来源、代码截图、飞书运行截图、HTML 展示截图、云服务器截图。
  • HTML 展示文件：替代 PPT，用浏览器打开即可展示。
  • 代码压缩包：包含 Python 脚本、README、requirements.txt、HTML 展示页，不包含 API Key。

报告里必须写清楚的点

  • 为什么采用“大盘—板块—个股—汇总”的多智能体结构。
  • 如何避免 AI 编造数据：先由脚本获取真实数据，再把数据交给模型总结；报告中保留数据来源字段。
  • 多方案对比：规则分析、单智能体分析、多智能体协同分析。
  • 遇到的问题：飞书权限、API Key 安全、服务器部署、JSON 格式不稳定等。

12. 常见问题与解决方法

问题解决方法
OpenClaw 安装失败先确认 Python 版本，换清华 pip 源，或按群公告安装命令重装。
飞书机器人没有回复检查机器人是否进群、事件订阅是否开启、回调地址是否正确、权限是否发布。
脚本运行路径找不到使用英文路径，避免中文目录；在项目根目录运行命令。
JSON 格式混乱脚本最后只 print 一个 JSON；模型总结另存字段，不要混在 stdout 里。
API Key 泄露风险使用 .env 保存，报告截图打码，Git 忽略 .env 文件。
现场网络不稳定准备 30 秒以上录屏，同时保留本地 HTML 日报作为备用展示。

最后检查清单

  • □ 三个人都能运行自己的 Claw。
  • □ 三个飞书机器人都在同一个群。
  • □ 三个触发词都能返回 JSON。
  • □ 汇总机器人能生成最终日报。
  • □ 有 HTML 展示页，不依赖 PPT。
  • □ 有云服务器部署截图。
  • □ 报告 PDF、HTML 展示文件、代码压缩包都准备好。
